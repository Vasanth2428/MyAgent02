# Beginner‑Friendly Change Report 📋

## What was added?

1. **Safety gate (`hitl_block_node.py`)**
   - A new agent node that looks at any file‑changing tool call (like `create_files`, `modify_files`, or `delete_file`).
   - If such a call is detected, the node pauses the workflow and asks the user for approval before the file is actually changed.
   - It does this by returning a **`Command`** that sets `waiting_for_approval` and stores the pending change in `pending_file_approvals`.

2. **Cross‑thread memory (`InMemoryStore`)**
   - Added an import of `InMemoryStore` in `src/graph/workflow.py`.
   - When the graph is compiled we now pass `store=InMemoryStore()`.  This gives us a persistent, in‑memory key‑value store that survives across different graph threads.
   - Use cases:
     - Cache AST fingerprints of files so the system can skip re‑parsing unchanged files (CODE‑02).
     - Keep long‑term “facts” about the codebase that survive after a thread finishes (MEM‑01).

3. **Streaming with LangGraph’s `values` mode**
   - Updated `src/core/engine.py` to use LangGraph’s **values streaming** instead of manually pulling the final state with `aget_state`.
   - The new loop:
     ```python
     async for change_dict in get_stream():
         yield {"event": "state_change", "state": change_dict}
         # handle waiting for approval …
         # handle supervisor updates …
     ```
   - Benefits:
     - Real‑time updates are pushed to the front‑end as soon as the state changes.
     - No extra “scrape the checkpoint at the end” step.
     - Keeps the graph’s blackboard clean and reduces state‑bloat.
